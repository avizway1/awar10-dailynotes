HDD : Hard Disk Drive : Mechanical Drive : uses spinning magnetic platters and a moving read and write head.

SDD : Solid State Drive : The Flash Drive : Data is going to store in micro chips.. i.e; USB

IOPS : Input and output Operations per Second.. 
--> HDD : 75 - 500 IOPS
--> SSD : 50k - 10L IOPS

Throughput : Data Trasfer Rate : MB/S , GbPS..

Throughpout = (IOPS x Data Block Size - KB) / 1024

10,000 x 4 / 1024 = 39.06 MB/s


lsblk -o NAME,PHY-SEC

cat /sys/block/nvme0n1/queue/physical_block_size

sudo fdisk -l

General Purpose SSD : Boot volume, general purpose workloads..  
Provisioned IOPS SSD : High performance
Throughout Optimised HDD : 
Cold HDD : 
Standard / magnetic : outdated..



Root volume : will run OS for our ec2 instance..

SSD 		: gp2, gp3, io1, io2
HDD			: sc1, st1
Magnetic	: Standard

root volume : gp2, gp3, io1,io2 & standard
Addl Volume : gp2, gp3, io1,io2 & standard.. St1 and sc1.. 

---

gp2 and gp3 : general purpose SSD : We can use this type for most common workloads.. 

gp2 : Min 1 Gb, Max 16 TB.. (for 1 gb, we get 3 IOPS (1:3 ratio) with min 100 IOPS) - 16,000 IOPS max..
gp3 : Min 1 GB, Max 64 TB.. (Min 3000 IOPS and Max 80,000 IOPS) - We can adjust IOPS based on app requirement, but cost us addl.

---

io1 and io2 : If our workload required specific IOPS count.. Critical workload required specific IOPS.. $$$$

io1 : min 4 gb.. Max: 16 TB.. (1 gb gives 50 IOPS (1:50 Ratio))
io2 : Min 4 GB.. Max: 64 TB.. (Min: 100 IOPS, Max: 200000 IOPS)

---

HDD : SC1, St1 : bit older as its HDD.. Less performance compared to SSD, but very cheap..

Throughput optimised HDD / st1 : Suitable for bigdata, datawarehousing and log processing.. 
Cold HDD / sc1 : Less performance and less cost than st1.. 

---

Standard / Magentic : Low cost storage Solutions.. Min 1 GB.. Max 1 TB..
